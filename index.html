develop 和 online k8s做分层业务时的区别总结

  • online和develop环境在使用Service时出现，dev可用， online不可用的情况,过程如下

    • namespace(wqctest) -> service(inner-servicelog) -> deployment（replicas 2）

  • 现在创建过程分别如下

    • dev创建过程 namespace(wqctest) -> service(inner-servicelog) -> deployment1 ; deployment1内的pod可访问inner-servicelog

    • online创建过程 namespace(wqctest) -> service(inner-servicelog) -> deployment1 ; 再别创建一个独立于service的deployment2。 deployment2内的pod可访问inner-servicelog

ceph使用过程问题集锦

2018-10-24

FAQ

1. 执行步骤：ceph-deploy new node 机器：centos 7.5 ceph Luminous版本 源：阿里云

2. 执行步骤 ：ceph-deploy disk zap node12:sdb

3.执行：ceph-deploy disk list node12

4.执行：[root@node12 my-cluster]# ceph-deploy disk zap node12 /dev/sdb

问题：[ceph_deploy][ERROR ] NeedDiskError: Must supply disk/path argument: node12:sdb

解决：在ceph luminous中创建bluestore的过程为指定data，block-db，block-wal

例如执行ceph-deploy osd create node1 –data /dev/sde –block-db /dev/sdf1 –block-wal /dev/sdf2

省略db 与wal的说明，只指定data则为ceph-deploy osd create node1 –data /dev/sdb则创建于同一个盘

问题： health: HEALTH_WARN
no active mgr

解决：Ceph Manager Daemon，简称ceph-mgr。 该组件的主要作用是分担和扩展monitor的部分功能，减轻monitor的负担，让更好地管理ceph存储系统ceph

ceph在 luminous中新加入了mgr功能模块，手动安装mgr即可

ceph-deploy mgr create node01 node02 node03

[root@node3 ~]# ceph osd pool delete ecpool ecpool –yes-i-really-really-mean-it

解决方法：
注：1-3步的操作必须在mon节点上执行

  1. 打开mon节点的配置文件：
    [root@node1 ceph]# vi /etc/ceph/ceph.conf
  2. 在配置文件中添加如下内容：
    [mon]
    mon allow pool delete = true
  3. 重启ceph-mon服务：
    [root@node1 ceph]# systemctl restart ceph-mon.target
  4. 执行删除pool命令：
    [root@node3 ~]# ceph osd pool delete ecpool ecpool –yes-i-really-really-mean-it
