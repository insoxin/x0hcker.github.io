利用nexus搭建maven私有仓库

2020-02-28

java

yum install java -y 
java --version

nexus

# 创建nexus用户，并设置该用户File Handle Limits
useradd nexus
echo "nexus - nofile 65536" >> /etc/security/limits.conf


wget https://sonatype-download.global.ssl.fastly.net/repository/repositoryManager/3/nexus-3.21.1-01-unix.tar.gz
tar -xzvf nexus-3.21.1-01-unix.tar.gz -C /data
mv /data/nexus-3.21.1 /data/nexus
mv /data/nexus* /opt/nexus
chown -R nexus:nexus /data/nexus /data/sonatype-work/


# 设置服务启动用户
echo 'run_as_user="nexus"' > /data/nexus/bin/nexus.rc

# 修改访问端口
/data/nexus/etc/nexus-default.properties
application-port=8081 


# 这里使用systemd管理服务
cat /etc/systemd/system/nexus.service

[Unit]
Description=nexus service
After=network.target

[Service]
Type=forking
ExecStart=/data/nexus/bin/nexus start
ExecStop=/data/nexus/bin/nexus stop
User=nexus
Restart=on-abort

[Install]
WantedBy=multi-user.target


# systemctl
systemctl daemon-reload
systemctl enable nexus.service
systemctl start nexus.service

k8s版本不同导致的clusterip不可用的解决方法

2018-12-04

develop 和 online k8s做分层业务时的区别总结

  • online和develop环境在使用Service时出现，dev可用， online不可用的情况,过程如下

    • namespace(wqctest) -> service(inner-servicelog) -> deployment（replicas 2）

  • 现在创建过程分别如下

    • dev创建过程 namespace(wqctest) -> service(inner-servicelog) -> deployment1 ; deployment1内的pod可访问inner-servicelog

    • online创建过程 namespace(wqctest) -> service(inner-servicelog) -> deployment1 ; 再别创建一个独立于service的deployment2。 deployment2内的pod可访问inner-servicelog

k8s deploy apiVersion 版本选择

  • develop 环境 k8s 1.11
